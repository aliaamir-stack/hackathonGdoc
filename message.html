01 · Features

7 AI Features

Every feature is impossible without AI — not enhanced by it. That's the judging criteria we're targeting.

01
🛡️

MedFact Shield

RAG pipeline over 5,000 PubMed abstracts. Decomposes any health claim into atomic sub-claims, retrieves evidence, returns a verdict with real paper citations. Destroys fake WhatsApp health forwards.

LangChain RAG · ChromaDB
02
🩺

Symptom Navigator

Clinical triage chatbot in Urdu + English. Multi-turn conversation with Redis memory. Forces structured output: urgency 1–5, ICD-10 differential, red flags, recommended action. Generates downloadable triage PDF.

ICD-10 · Pydantic · reportlab
03
💊

Medicine Scanner

Photo → OpenCV preprocessing → Tesseract OCR → Gemini Vision identification → OpenFDA official data → DrugBank interaction check. Flags expired packaging. Works in under 4 seconds.

Vision AI · OCR · OpenFDA
04
🗺️

Outbreak Radar

Every symptom report logs anonymized lat/lng. Celery runs DBSCAN every 15 min. Prophet detects time-series anomalies. Results broadcast via WebSocket to a live Leaflet heatmap. WHO RSS feed overlaid.

DBSCAN · Prophet · WebSocket
05
🚨

Emergency First Aid

Voice-activated. 15 protocol JSONs cached offline via Service Worker. Web Speech API transcribes → Gemini matches protocol → Web Speech TTS guides step-by-step. Telegram bot sends GPS to contacts.

Voice · PWA offline · Telegram
06
🏥

Resource Locator

Triage result feeds directly into facility search. Urgency 5 → filters for trauma centres. OSRM open-source routing. 3,000+ OSM facilities. Blood bank search. Nearest AED within 500m.

OSRM · PostGIS · Leaflet
07
📊

Health Dashboard

JWT-protected. Pakistan district choropleth map. Timeline scrubber to replay 30-day outbreak evolution. Neighbourhood health score = weighted outbreak proximity + facility access + report volume. FHIR R4 export.

GeoJSON · Recharts · FHIR

Three Rules

Violating any of these will cost you hours at the worst possible time.

Rule 1 — Never push broken code to dev.

Your personal branch is your sandbox — break things there freely. Only push to dev when your feature runs without crashing. One broken merge blocks all 6 people.

Rule 2 — M2 and M6 own all merges into dev.

Nobody else touches dev directly. If you're ready to merge, message M2 or M6 in your group chat. They do it. This prevents two people merging simultaneously and creating a conflict war at 3 AM.

Rule 3 — Every dev merge is immediately followed by everyone pulling.

The moment dev updates, message the group "dev updated — pull now". Everyone runs git merge dev before their next commit. Skipping a sync is how you get a 200-line conflict at 7 AM.
